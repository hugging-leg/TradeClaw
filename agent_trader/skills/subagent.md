# SubAgent — 并行分析

使用 `spawn_parallel_subagents` 或 `spawn_subagent` 将分析任务拆分给子 Agent 并行执行。

## 使用时机

- 需要同时分析多个维度（技术面、基本面、情绪面、宏观面）
- 需要同时分析多只股票
- 需要同时搜索多个不同主题的信息
- 任何可以并行化的独立分析任务

**不要在主流程中串行地逐个收集信息** — 如果有多个独立的分析维度，应该拆分给子 Agent。

## 并行分析（推荐）

```
spawn_parallel_subagents(
    tasks=[
        "分析 NVDA 的技术面：查看近期价格走势、RSI、MACD 等技术指标，给出技术面评估",
        "分析 NVDA 的基本面：查看最新财报、营收增长、估值水平，给出基本面评估",
        "搜索 NVDA 相关的最新新闻和市场情绪，评估市场对该股的态度",
    ],
    timeout_seconds=180,
)
```

## 单个子Agent

```
spawn_subagent(
    task="深入分析美联储最新政策声明对科技股的影响",
    timeout_seconds=120,
)
```

## 子Agent 能力

- 拥有与你相同的**只读工具**（行情、新闻、搜索、代码执行、记忆等）
- **不能执行交易**（rebalance_portfolio 等被排除）
- **不能创建调度**（schedule_next_analysis 被排除）
- **支持多层递归**：子 Agent 可以再派生子 Agent，受 `subagent_max_depth` 限制（默认 2 层）
  - depth=0 为主 Agent，depth=1 为子 Agent，depth=2 为子子 Agent
  - 当 depth+1 >= max_depth 时，子 Agent 不再拥有 spawn 工具

## 最佳实践

1. **任务描述要具体**：告诉子 Agent 要分析什么、用什么工具、输出什么格式
2. **合理拆分**：每个子 Agent 聚焦一个独立维度，避免重叠
3. **汇总决策**：子 Agent 的结果返回后，由你做最终综合判断和交易决策
4. **控制数量**：通常 2-5 个并行子 Agent 最优，过多会增加延迟
5. **设置超时**：复杂分析可适当增加 timeout_seconds
6. **谨慎递归**：仅在任务确实复杂、需要进一步分解时才让子 Agent 再派生子 Agent，避免不必要的 token 消耗

## 典型模式

### 投资组合全面分析
```
tasks = [
    "获取当前持仓信息，分析各持仓的集中度和相关性风险",
    "搜索最新的宏观经济新闻（美联储政策、就业数据、通胀数据），评估宏观环境",
    "搜索持仓股票的最新公司新闻和行业动态，评估个股风险",
    "分析持仓股票的技术面（价格趋势、支撑阻力位），给出技术评估",
]
```

### 个股深度分析
```
tasks = [
    "分析 AAPL 的技术指标和价格走势",
    "分析 AAPL 的基本面（财报、估值、增长）",
    "搜索 AAPL 的最新新闻和分析师评级",
]
```
